# PoE Tools: объединение Exile Leveling и русских regex-генераторов

Статус: дизайн согласован 2026-07-21.

## Цель

Превратить `exile-leveling` в единый русскоязычный сайт **PoE Tools**. Существующая страница «Полезное» становится главной витриной, прокачка сохраняется как один из основных инструментов, а все рабочие RU/EN regex-генераторы из `poe-regex-ru` переносятся в тот же web-продукт.

Результат считается завершённым только после production-деплоя GitHub Pages, внешнего smoke-теста опубликованного URL и подтверждённого перехода со старого сайта.

## Канонический репозиторий и границы

Канонический репозиторий — `Simonerrror/exile-leveling`. Его workspace-структура `common`, `seeding`, `web` сохраняется.

Входит в работу:

- общая оболочка PoE Tools;
- главная страница-каталог на базе текущего раздела «Полезное»;
- существующие прокачка, дерево, камни, редакторы и полезные ресурсы;
- все двенадцать пользовательских regex-маршрутов из `poe-regex-ru`;
- перенос regex domain/data/test слоя без старой Create React App оболочки;
- унификация RU/EN, профилей и локального состояния;
- удаление фильтров связей сокетов по цветам;
- оптимизация стартовой загрузки и production-сборки;
- GitHub Pages deploy и cutover.

Не входит:

- новые игровые инструменты, которых нет ни в одном из двух проектов;
- OAuth, чтение живого стэша, desktop companion, OCR и overlay;
- смена Vite/React/workspace-стека `exile-leveling`;
- новая UI-библиотека или runtime-зависимость только ради редизайна.

## Информационная архитектура

Основные маршруты:

- `#/` — главная «Полезное»;
- `#/leveling` — текущий маршрут прокачки;
- `#/regex` — каталог regex-генераторов;
- `#/regex/vendor` — закуп у торговцев;
- `#/regex/maps` — модификаторы карт;
- `#/regex/items` — крафт предметов;
- `#/regex/mapnames` — названия карт;
- `#/regex/expedition` — экспедиция;
- `#/regex/heist` — кража;
- `#/regex/flasks` — флаконы;
- `#/regex/beast` — бестиарий;
- `#/regex/tattoo` — татуировки;
- `#/regex/runegraft` — рунограммы;
- `#/regex/scarab` — скарабеи;
- `#/regex/jewel` — самоцветы.

Существующие маршруты дерева, камней, сборки и редакторов сохраняются. `#/useful` перенаправляется на новый корень `#/`, а прокачка получает постоянный адрес `#/leveling`. Старый bookmark `#/` откроет новую главную, где «Продолжить прокачку» и карточка прокачки являются первыми действиями. Остальные существующие deep links не должны превращаться в пустую страницу.

Общая навигация содержит PoE Tools, «Полезное», «Прокачка», «Regex», переключатель языка и компактное overflow-меню. Навигация `poe-regex-ru` не переносится.

## Главная «Полезное»

Композиционный референс — каталог Linear Integrations; палитра и плотность — Exile Leveling. Не копируются фирменные ассеты, тексты или компоненты Linear.

На широком экране слева находится компактный список категорий, справа — двухколоночная сетка. На узком экране категории превращаются в горизонтальные фильтры, сетка — в одну колонку.

Секции:

1. **Продолжить** — текущий акт и шаг прокачки, если сохранён прогресс.
2. **Основные инструменты** — прокачка и общий каталог regex.
3. **Regex** — входы в тематические группы или отдельные генераторы.
4. **Справочники** — дерево, камни и связанные внутренние инструменты.
5. **Внешние ресурсы** — существующий проверенный каталог ссылок.
6. **Недавние** — локально сохранённые последние внутренние инструменты; секция скрыта при отсутствии истории.

Поиск фильтрует внутренние инструменты и внешние ресурсы на месте. Карточка целиком является ссылкой, содержит небольшую иконку, название и содержательное описание максимум в две строки.

## Визуальная система

Базовые токены сохраняются из `web/src/index.css`:

- основной фон `#161616`;
- рабочая поверхность `#222222`;
- основной текст `#d4d4d4`;
- усиленный текст `#f5f5f5`;
- hover/active `#242424` и `#404040`;
- нейтральные акценты `#808080` и `#505050`;
- граница 1 px;
- базовый радиус 4 px;
- Source Sans Pro с системным fallback.

Не используются декоративные hero-блоки, стекло, blur, glow, крупные фоновые картинки и тяжёлые тени. Цвет применяется семантически к тексту, тонкой границе и состоянию взаимодействия, а не к большим заливкам.

Игровые роли переиспользуют семантику Exile Leveling:

- элитные противники и боссы — `darkorange`;
- области — `#ffcf91`;
- задания — `gold`;
- важный quest action — `lime`;
- порталы и точки перемещения — `#8ab3ff`;
- испытания — `darkseagreen`;
- уровень и вторичная информация — `#808080`.

Категории главной используют существующие акценты:

- regex/калькуляторы — `#d6a85f`;
- планирование/прокачка — `#68a8d8`;
- торговля — `#7ebc87`;
- аналитика — `#b68ad8`;
- прочее — `#d8796f`.

Цвет не может быть единственным носителем значения: статус сопровождается текстом, иконкой или формой.

## Компоновка regex-инструмента

На desktop страница содержит breadcrumb и короткий заголовок, затем две области:

- основная колонка с поиском, массовыми действиями и фильтрами;
- узкая липкая колонка результата с regex, длиной, статусом и действиями.

Секции фильтров разделяются заголовком и тонкой линией. Многослойные декоративные карточки не используются. Пользовательский текст, автокопирование и редко используемые параметры находятся под disclosure «Дополнительно».

На mobile фильтры идут одним потоком. Результат становится свёрнутой липкой нижней панелью с длиной и основной кнопкой «Копировать»; панель разворачивается по нажатию и не перекрывает последний элемент формы.

Состояния результата:

- безопасный результат;
- предупреждение о приближении к лимиту;
- превышение 250 символов;
- безопасный двухпроходный A/B fallback;
- отсутствие данных;
- обнаруженная коллизия;
- ошибка Clipboard API.

Технический regex отображается моноширинным нейтральным текстом. Безопасность, предупреждение и ошибка используют зелёный, золотой и красный статусы с текстовой подписью.

## Связи сокетов

Фильтры и сохранённые настройки связей по комбинациям цветов удаляются полностью. Пользователь выбирает только количество:

- не важно;
- 4 связи;
- 5 связей;
- 6 связей.

Миграция старого профиля сохраняет выбранное количество связей, если оно известно, и игнорирует цветовую комбинацию. Отсутствующее или некорректное значение становится «не важно». Миграция идемпотентна и не изменяет остальные настройки профиля.

## Архитектура кода и данных

`exile-leveling` остаётся Vite/React workspace-проектом.

- чистые regex-каталоги, query plan, compiler, optimizer, fixtures и миграции размещаются в `common`;
- route-level UI и адаптеры размещаются в `web/src/tools/regex`;
- обновление и валидация generated data размещаются в `seeding`;
- каждый regex-маршрут загружается через `React.lazy`;
- данные каждого домена публикуются отдельным JSON-чанком и загружаются только его маршрутом;
- RU и EN являются поддерживаемыми runtime-языками; таблицы прочих языков из старого проекта не входят в PoE Tools;
- общие каталоги не дублируются между TypeScript bundle и `public` JSON;
- последний known-good data manifest остаётся воспроизводимым и версионированным.

Runtime-ошибка чанка или данных перехватывается существующим ErrorBoundary и показывает понятный retry, а не пустую страницу.

## Производительность и размер

Измеренный baseline `poe-regex-ru`:

- `GeneratedRussianItems.ts` — 5.1 MB исходного содержимого;
- `GeneratedItemMods.ts` — 3.3 MB;
- остальные generated-каталоги — около 1.4 MB;
- основной production JS — 16.1 MB raw / 1.46 MB gzip;
- отдельные economy JSON — около 2.8 MB;
- production без sourcemaps — около 19 MB;
- полная папка build с sourcemaps — около 34 MB.

Причина — generated data как TypeScript-объекты, статические импорты всех маршрутов, ненужные языковые таблицы и публикация sourcemaps.

Обязательные бюджеты:

- стартовый JS главной не более 250 KB gzip;
- код и данные regex не загружаются до перехода в regex-раздел;
- отдельный маршрут не может молча добавить generated catalog в общий entry chunk;
- production sourcemaps не публикуются в GitHub Pages;
- размер основных чанков и статических данных публикуется в CI-отчёте;
- превышение стартового бюджета блокирует deploy.

Новые изображения должны быть SVG, WebP или AVIF и иметь пользовательскую функцию. Неиспользуемые upstream-ассеты не переносятся.

## Этапы миграции

### 1. PoE Tools shell

- переименовать продукт в PoE Tools;
- сделать «Полезное» корневым маршрутом;
- перенести прокачку на `#/leveling`, сделать `#/` новой главной и перенаправить `#/useful` на `#/`;
- обновить общую навигацию;
- внедрить каталог главной и общие токены;
- сохранить все существующие функции Exile Leveling.

### 2. Regex core

- перенести domain-код и минимально необходимые generated источники в `common`/`seeding`;
- исключить языки, не поддерживаемые PoE Tools;
- перевести большие runtime-каталоги из TypeScript в domain JSON;
- перенести fixtures и parity-проверки;
- добавить миграцию socket-link настроек.

### 3. Regex UI

Первый вертикальный срез: карты. Затем предметы и торговцы. После проверки общих компонентов переносятся флаконы, названия карт, экспедиция, кража, звери, татуировки, рунограммы, скарабеи и самоцветы.

Каждый маршрут считается перенесённым только после parity-проверки результата, RU/EN smoke и подтверждения lazy chunk boundary.

### 4. Cutover

- выполнить полный typecheck, тесты, build и bundle report;
- проверить desktop/mobile, keyboard/focus и clipboard;
- опубликовать GitHub Pages из `exile-leveling`;
- проверить публичный URL и deep links;
- добавить в `poe-regex-ru` архивное сообщение и ссылку на PoE Tools;
- архивировать старый репозиторий только после успешного production smoke.

## Проверки

Domain/data:

- RU/EN semantic parity;
- exact и optimized equivalence;
- catalog collision scan;
- numeric boundaries;
- лимит 250 и A/B fallback;
- deterministic generated artifacts;
- socket-link migration и отсутствие цветовых опций.

Web:

- корневой каталог и поиск;
- продолжение прокачки;
- все внутренние маршруты и redirects;
- выбор фильтров, сброс, копирование и disclosure;
- error/retry при недоступном data chunk;
- keyboard/focus и mobile sticky result;
- локализация RU/EN.

Release:

- чистый deterministic install из lockfile;
- typecheck и production build;
- bundle budget;
- отсутствие production sourcemaps;
- GitHub Pages deployment;
- HTTP 200 для entry/assets и ручной smoke всех основных маршрутов.

## Критерии завершения

Работа завершена, когда:

1. Главная PoE Tools опубликована и является каталогом «Полезное».
2. Прокачка и существующие инструменты Exile Leveling не потеряли функций.
3. Все двенадцать regex-маршрутов доступны из того же приложения.
4. Цветовые комбинации связей отсутствуют в UI, state и generated output; количество 4/5/6 работает и мигрируется.
5. RU/EN parity, regex fixtures, typecheck, build и bundle budget проходят.
6. Стартовый JS главной не превышает 250 KB gzip.
7. GitHub Pages публично отдаёт приложение и его deep links без пустой страницы.
8. Старый `poe-regex-ru` содержит понятный переход и архивируется после cutover.

## Риски и решения

- **Большой одномоментный перенос.** Использовать вертикальные срезы и не переносить несколько генераторов до проверки общего пути на картах.
- **Различие React/TypeScript/test стеков.** Переносить чистую логику, а не старую UI-оболочку; адаптировать тесты к существующему workspace без новой runtime-зависимости.
- **Скрытый рост entry bundle.** Проверять chunk graph и gzip-бюджет в каждом этапе.
- **Поломка старых ссылок и localStorage.** Явные redirects и идемпотентные миграции с fixtures.
- **Устаревшие economy fixtures.** Сохранять последний known-good manifest и не блокировать загрузку оболочки из-за внешнего API.
- **Архивация до готовности.** Не архивировать `poe-regex-ru`, пока production smoke PoE Tools не завершён.
